## 4. תרחישי שימוש (Use Cases)

### 4.1 UC-1: משתמש צופה במוצרים ומקבל המלצות

**שחקנים:** משתמש מחובר  
**תנאי התחלה:** המשתמש מחובר  
**זרימה עיקרית:**
1. המשתמש נכנס לדף הבית
2. המערכת מציגה "מומלץ בשבילך" ו"מוצרים פופולריים"
3. המשתמש לוחץ על מוצר
4. המערכת מציגה דף מוצר + "מוצרים דומים"
5. המערכת רושמת צפייה

### 4.2 UC-2: Admin מוסיף מוצר חדש

**שחקנים:** Admin  
**תנאי התחלה:** Admin מחובר  
**זרימה עיקרית:**
1. Admin נכנס לממשק ניהול
2. Admin מוסיף מוצר (שם, תיאור, מחיר, תמונה)
3. המערכת שולחת ל-ML Service לקטגוריזציה
4. ML מחזיר קטגוריה ותת-קטגוריה
5. המערכת שומרת את המוצר עם הקטגוריה

### 4.3 UC-3: הרצת Phase 1

**שחקנים:** מנהל מערכת / Admin  
**תנאי התחלה:** Backend ו-Python scripts זמינים  
**זרימה עיקרית:**
1. מנהל קורא POST /api/ml/phase1
2. המערכת מייצאת מוצרים ל-CSV
3. המערכת מריצה product_categorization.py, user_categorization.py
4. המערכת קוראת תוצאות ועדכנת את ה-DB
5. רק מוצרים/משתמשים ללא קטגוריה מתעדכנים

### 4.4 UC-4: אורח צופה במוצרים

**שחקנים:** אורח  
**תנאי התחלה:** אין  
**זרימה עיקרית:**
1. אורח נכנס לדף הבית
2. המערכת מציגה "מוצרים פופולריים" והמלצות כלליות
3. אורח יכול לדפדף, לחפש, לראות מוצרים
4. אורח לא יכול להוסיף לעגלה (נדרש רישום/התחברות)

## 5. תלויות ומגבלות

### 5.1 תלויות חיצוניות

| רכיב | גרסה | הערות |
|------|-------|-------|
| PostgreSQL | 14+ | Docker |
| RabbitMQ | 3.x | אופציונלי |
| Elasticsearch | 8.x | חיפוש |
| Docker | Desktop | להרצת תשתית |

### 5.2 מגבלות ידועות

- Phase 1 דורש Python וקבצי נתונים (datasets/raw/).
- ML Service מופעל אוטומטית עם Backend אך יכול לרוץ בנפרד.
- המלצות מבוססות על נתונים ב-DB; ללא אינטראקציות – ההמלצות כלליות.
