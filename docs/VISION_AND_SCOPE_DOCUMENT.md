# מסמך חזון ותכולה (Vision and Scope Document)
## מערכת E-Commerce עם Machine Learning

---

## 1. מבוא

### 1.1 מטרת המסמך
מסמך זה מגדיר את החזון, היקף התכולה והגבולות של פרויקט מערכת E-Commerce חכמה המשלבת Machine Learning להמלצות מוצרים וקטגוריזציה אוטומטית.

### 1.2 קהל יעד
- מנהלי הפרויקט
- מפתחים
- מנחים אקדמיים
- בעלי עניין בפרויקט

---

## 2. רקע והקשר

### 2.1 הבעיה
חנויות E-Commerce מתמודדות עם:
- **קטגוריזציה ידנית** – תיוג מוצרים לקטגוריות דורש זמן רב וטעויות אנוש.
- **המלצות גנריות** – ללא למידת מכונה, ההמלצות אינן מותאמות אישית.
- **ניהול מלאי** – מוצרים חדשים זקוקים לקטגוריה לפני שהם מוצגים בצורה נכונה.

### 2.2 ההזדמנות
שילוב ML מאפשר:
- **קטגוריזציה אוטומטית** של מוצרים ומשתמשים.
- **המלצות מותאמות אישית** על בסיס התנהגות המשתמש.
- **מוצרים דומים** על בסיס קטגוריה ותוכן.

---

## 3. חזון (Vision)

### 3.1 הצהרת החזון
**"מערכת E-Commerce חכמה שמשתמשת ב-Machine Learning כדי לספק חוויית קנייה מותאמת אישית, קטגוריזציה אוטומטית של מוצרים ומשתמשים, והמלצות רלוונטיות."**

### 3.2 יעדים עיקריים
1. **קטגוריזציה אוטומטית** – מוצרים ומשתמשים מקבלים קטגוריות על בסיס ML (XGBoost, Random Forest).
2. **מערכת המלצות היברידית** – שילוב Collaborative Filtering, Content-Based Filtering ו-Neural Network Ranking.
3. **ממשק משתמש מלא** – Frontend, Backend, Admin לניהול מוצרים.
4. **אינטגרציה מלאה** – ML מופעל בזמן אמת (single-item) ובאצווה (Phase 1).

---

## 4. היקף התכולה (Scope)

### 4.1 תכולה כלולה (In Scope)

| רכיב | תיאור |
|------|-------|
| **Backend** | Spring Boot, PostgreSQL, RabbitMQ, Elasticsearch |
| **Frontend** | React – דפי מוצרים, עגלה, פרופיל, חיפוש |
| **Admin** | ניהול מוצרים, עריכת קטגוריות, סנכרון ML |
| **ML Phase 1** | קטגוריזציה של מוצרים (XGBoost) ומשתמשים (Random Forest) |
| **ML Phase 2** | מערכת המלצות היברידית (Collaborative + Content-Based + Neural Network) |
| **ML Phase 3** | קטגוריזציה של פריט בודד (מוצר/משתמש) |
| **שירות ML (Flask)** | API מהיר לקטגוריזציה ולהמלצות |
| **סנכרון DB** | סנכרון חד-פעמי מקבצי CSV (seed, Phase 1) |

### 4.2 תכולה לא כלולה (Out of Scope)

| רכיב | הסבר |
|------|------|
| תשלומים אמיתיים | Stripe מוזכר אך לא חלק מהדרישות המרכזיות |
| אפליקציית מובייל | רק Web |
| ריבוי שפות | תמיכה בשפה אחת (עברית/אנגלית כפי שמוגדר) |
| פיתוח אלגוריתמי ML חדשים | שימוש במודלים קיימים (XGBoost, Random Forest, TF-IDF) |

---

## 5. גבולות וקבלת החלטות

### 5.1 גבולות פונקציונליים
- המערכת תומכת במשתמשים רשומים ובאורחים.
- ההמלצות מבוססות על רכישות, צפיות ועגלה.
- קטגוריזציה של מוצר חדש מתבצעת אוטומטית בעת הוספה/עדכון.

### 5.2 הנחות (Assumptions)
- יש גישה לנתוני דמו (מוצרים, משתמשים, אינטראקציות).
- השרתים (PostgreSQL, RabbitMQ, Elasticsearch) רצים ב-Docker.
- Python 3.7+ ו-Java 17+ זמינים לסביבת הפיתוח.

### 5.3 אילוצים (Constraints)
- פרויקט אקדמי – התמקדות בלימוד טכנולוגיות.
- זמן מוגבל – פיתוח בשלבים (Phases).

|--------|-------|

| צוות הפיתוח | מימוש, בדיקות |

## 7. סיכום

מערכת E-Commerce חכמה זו משלבת Backend (Spring Boot), Frontend (React), ושירות ML (Python/Flask) כדי לספק קטגוריזציה אוטומטית והמלצות מותאמות אישית. היקף התכולה ממוקד ב-Phases 1–3 של ה-ML, ניהול מוצרים, עגלת קניות וחיפוש. מסמכים נוספים (SDD, SRS) מפרטים את העיצוב והדרישות בפירוט.
